CMIG Workflow Tutorial

Installation and scenario-based guide for model-pool search, strain growth profiling, abundance impact sweeps, host-microbe coupling, and dynamics.

1. Installation

CMIG expects user-prepared GEM files. It does not download or curate AGORA, VMH, Recon, Human-GEM, or BiGG model collections.

git clone https://github.com/jyryu3161/CMIG.git
cd CMIG
uv sync --extra engine --extra gui --extra render --extra stats

Verify the environment:

uv run cmig version
uv run cmig solvers
uv run ruff check cmig tests
uv run mypy cmig
uv run pytest -q
The current product workflows assume Gurobi for full LP/QP community analysis, host-microbe coupling, search, and FVA-oriented tasks.

2. Launch The GUI

uv run python - <<'PY'
from PySide6.QtWidgets import QApplication
from cmig.gui.app import build_main_window

app = QApplication([])
window = build_main_window(lang="en")
window.resize(1500, 950)
window.show()
app.exec()
PY

3. Prepare Model Folders

Place microbial SBML/XML models in one folder:

models/
  iHN637.xml
  iML1515.xml
  other_strain.xml

If each strain has its own subfolder, pass --recursive in the CLI. The GUI product search currently uses the selected folder directly.

4. Scenario A: Find The Best Model Combination

Question: which 2-member combination produces the most butyrate?

uv run cmig search \
  --model-dir models \
  --target but \
  --min-size 2 \
  --max-size 2 \
  --top-k 10 \
  --strategy auto \
  --out runs/search_butyrate

Review search_rankings.csv, search_member_matrix.csv, search_plot.svg, and search_scatter.svg.

5. Scenario B: Estimate Strain-Specific Growth

Question: how does each model grow alone, and how does it grow after being placed in the full community?

uv run cmig strain-growth \
  --model-dir models \
  --out runs/strain_growth
FieldMeaning
single_growthFBA growth of the individual GEM.
community_member_growthMember growth after MICOM community construction and cooperative tradeoff.
abundanceCommunity abundance used for that member.

In the GUI, open the Search tab, choose the model folder, and click Strain Growth.

6. Scenario C: Increase One Strain Ratio

Question: if one strain becomes more abundant, does it have more influence on acetate exchange or community growth?

uv run cmig abundance-impact \
  --model-dir models \
  --member iML1515 \
  --fractions 0.1,0.25,0.5,0.75 \
  --target ac \
  --out runs/iML1515_ac_ratio

CMIG sets the selected member to each requested fraction and rescales the other members to fill the remaining abundance. MICOM then recomputes growth and fluxes for each condition.

OutputUse
abundance_impact.csvMain sweep table for growth, target flux, and influence share.
member_growth_by_abundance.csvMember-level growth and realized abundance per sweep point.
abundance_impact_plot.svgQuick visual check of growth, exchange flux, and target share.
Interpret this as a controlled sensitivity analysis. A higher abundance can increase a member's contribution to a target flux, but the result still depends on medium, objective, model constraints, and cross-feeding.

In the GUI, open the Search tab, enter a member id in the Growth/Ratio member box, set fractions, and click Ratio Impact.

7. Scenario D: Host-Microbe Coupling

Question: which microbial secretion can be directly consumed by a Recon/Human-GEM style host model?

uv run cmig host-microbe-bigg \
  --host models_human/Recon3D.xml \
  --model-dir models \
  --out runs/host_microbe

This workflow assumes compatible BiGG-style exchange identifiers. CMIG does not perform Recon-specific curation or automatic host interface mapping.

8. Scenario E: Dynamics

Run single-model well-mixed dFBA with the default aerobic preset:

uv run cmig dfba \
  --model models/iML1515.xml \
  --dt 0.1 \
  --out runs/dfba_iML1515

The default tracked exchanges are glucose, oxygen, acetate, and D-lactate when present in the model.

9. LLM and Automation Entry Point

Agents that need to run CMIG without the desktop GUI should first request the workflow map:

uv run cmig workflows --format json

The JSON maps each working GUI surface to the equivalent CLI command, required arguments, common options, key output files, and a copyable example. After a run completes, inspect the output directory with:

uv run cmig inspect-run --run-dir runs/search_butyrate --format json

This reports the detected run kind, status, run hash when available, summary keys, and artifacts. It is the CLI counterpart to opening a run in the GUI Profile/Open Run view.

10. Recommended Review Checklist

  1. Confirm every model imports cleanly in pool_diagnostics.csv.
  2. Check whether the target exchange exists in candidate models.
  3. Compare single_growth and community_member_growth.
  4. For ratio sweeps, inspect both target_influence_share and community_growth; do not optimize influence alone.
  5. Export SVG figures for inspection, then use the CSV files for final manuscript plotting if journal-specific styling is required.
